# Code of Conduct

## Purpose

LowHunt is maintained as a professional, respectful project for public-data-only OSINT, defensive investigation, and authorized research. Everyone participating in this repository is expected to collaborate in a way that is constructive, lawful, and safe for others.

## Relationship to License

This Code of Conduct governs community behavior.

The project `LICENSE` governs usage, redistribution, modification, attribution, and permission boundaries for LowHunt.

Nothing in this Code of Conduct grants additional rights beyond the `LICENSE`, and nothing here overrides the `LICENSE`.

## Expected Behavior

Examples of behavior that support this project include:

- being respectful and constructive in issues, pull requests, reviews, and discussions
- focusing feedback on code, design, testing, and documentation rather than on the person
- being patient with learning-focused questions and beginner contributions
- respecting the project's public-data-only and authorized-use boundaries
- following maintainer guidance on repository hygiene, scope, and release quality
- helping keep the project safe, readable, and useful for defensive and investigative work

## Unacceptable Behavior

Examples of behavior that are not acceptable include:

- harassment, threats, hate speech, or demeaning language
- personal attacks, trolling, or deliberate disruption
- sharing private information without permission
- encouraging, promoting, or normalizing unauthorized testing, abuse, or illegal activity
- submitting content intended to add protection bypass, credential abuse, or intrusive behavior outside the project scope
- misrepresenting authorship, intentionally removing attribution, or attempting to bypass project governance

## Scope

This Code of Conduct applies across all project spaces, including:

- issues
- pull requests
- code review comments
- discussions
- repository documentation
- security or bug reports submitted through project contact channels

## Reporting

If you experience or witness behavior that violates this code of conduct, report it to the maintainer through the repository contact channels.

Contact:

- `voltsparx@gmail.com`

## Enforcement

Project maintainers may take any action they consider appropriate to protect the health of the project, including:

- warnings
- comment or content removal
- temporary restrictions
- permanent removal from project spaces

Maintainers may also reject contributions that are abusive, unlawful, clearly out of scope, or incompatible with the project's safety and usage boundaries.
